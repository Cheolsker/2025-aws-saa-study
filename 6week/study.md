
# 1번
## 키워드
- 데이터속도: 10.8GB/시간 X 24시간/일 = 259.2GB/일
- 필요한 전송기간: 2314일 (600000GB / 259.2GB)

- 2주 이내에 데이터 전송 완료
- 데이터는 전송중 암호화
- 비용 효율적이어야.

정답 > C


## 근거
- A 
  - HTTPS는 암호화를 제공하지만, 데이터는 여전히 100Mbps 인터넷 링크를 통해 전송
  - 시간 제약(>6년 소요)을 충족할 수 없음

- B
  - VPN은 암호화를 제공하지만, 기본적으로 인터넷 연결(100Mbps)에 의존. 
  - 시간 제약을 충족할 수 없으며, VPN 트래픽은 일반적으로 오버헤드 때문에 속도가 더 느려짐

- C
  - Snowball Edge 장치는 물리적 장치를 통해 데이터를 전송하여 네트워크 속도 제약을 우회. 
  - 장치당 80TB의 유효 전송 용량을 제공, 7~8개의 장치를 사용하여 2주 이내에 600TB를 쉽게 전송할 수 있음. 
  - 데이터는 전송 중 및 장치에 저장될 때 암호화됨.
  - 이는 대용량 데이터를 빠르고 안전하게 전송하는 데 있어 가장 비용 효율적인 방법

- D
  - 10Gbps Direct Connect를 설정하는 데는 막대한 초기 비용과 월별 반복 비용이 발생하며, 설치에 수 주가 소요될 수 있음.
  -  이 회사는 일회성 전송만 필요한데, Direct Connect는 영구적이고 일관된 고대역폭 연결이 필요할 때 적합
  - 이 시나리오에서는 시간과 비용 측면에서 모두 비효율적



# 2번
## 키워드
- 데이터가 자주 변경된다.
- 데이터의 모든 수준에서 감사 액세스가 필요
  - 데이터 CRUD
- 온프레미스의 데이터를 AWS로 마이그레이션

정답 > A

## 근거
- A
  - DataSync는 대규모 데이터 마이그레이션을 위한 온라인 전송 서비스
  - NAS와 같은 온프레미스 시스템에서 Amazon S3로 데이터를 안전하고 효율적으로 전송하는 데 최적화

- B
  - Snowcone은 네트워크 속도가 느릴 때 사용되는 물리적 전송 장치이며, 데이터가 자주 변경되는 환경에는 부적합
  
- C
  - S3 Transfer Acceleration는 먼 거리에서 S3로의 인터넷 전송 속도를 높임
  - 온프레미스 NAS에서 S3로의 마이그레이션을 위한 전용 도구는 아님

- D
  - Storage Gateway는 온프레미스 환경에 AWS 스토리지를 확장하거나 연결하는 솔루션
  - 대규모 일회성 마이그레이션을 위한 주된 도구는 아님

- AWS CloudTrail
  - AWS 계정 내의 관리 및 데이터 활동을 기록하고 감사하는 데 사용되는 서비스
  - 관리 이벤트 (Management Events)
    - S3 버킷 생성, IAM 사용자 생성 등 리소스에 대한 제어 평면(Control Plane) 작업 기록
  - 데이터 이벤트 (Data Events)
    - S3 객체 업로드(Put Object), 다운로드(GetObject) 등.. 리소스 내의 데이터 평면(Data Plane) 작업 기록.


# 3번
## 키워드
- 게임 플랫폼 보유, 지연시간에 민감
- ALB 뒤에 자동확장그룹의 일부인 EC2에서 실행

정답 > A

## 근거
- A
  - AWS Global Accelerator
    - 사용자의 트래픽을 일반 인터넷 대신 AWS 글로벌 네트워크 엣지 로케이션을 통해 가장 가까운 AWS 지역으로 라우팅함으로써 지연 시간을 크게 줄이는 데 최적화된 서비스
    - 안정적인 고정 IP 주소를 제공하며, TCP 및 UDP와 같은 프로토콜을 사용하는 지연 시간에 민감한 게임 트래픽에 가장 적합
    - 구성된 엔드포인트 그룹 내의 각 엔드포인트(ALB)에 대해 **상태 확인 (Health Check)**을 수행
    - 특정 지역의 ALB가 비정상으로 감지되면, Global Accelerator는 자동으로 해당 엔드포인트로의 트래픽 흐름을 중단하고 가장 가까운 다른 정상적인 지역 엔드포인트로 트래픽을 즉시 리디렉션

- B, C
  - Amazon CloudFront 
    - 웹 트래픽(HTTP/HTTPS) 및 정적 콘텐츠의 캐싱을 위해 설계
    - 지연 시간에 민감한 실시간 게임 트래픽 (종종 UDP 사용)에 대한 최적화 기능은 Global Accelerator가 월등
    - 동적 애플리케이션에 부적합한 S3를 오리진으로 지정함.

- D
  - DynamoDB 및 DAX
    - 데이터베이스 계층의 읽기 성능을 개선하는 솔루션
    - 네트워크 지연 시간을 줄이고 전역 트래픽을 분산하며 엔드포인트 상태를 모니터링하는 라우팅 문제와는 관련이 없음


# 4번
## 키워드
- AWS RDS가 있는 EC2 인스턴스에 애플리케이션 배포
- 최소 권한 원칙을 사용하여 데이터베이스 액세스 자격증명 구성
- SQL 주입 및 웹 기반 공격으로부터 애플리케이션 및 데이터베이스 보호하려고 함

정답 > B

## 근거
- A
  - 기본적인 네트워크 접근 제어를 제공할 뿐
  - SQL 주입이나 XSS와 같은 애플리케이션 계층(Layer 7)의 정교한 공격을 탐지하거나 방어할 수 없음.
- B
  - AWS WAF (Web Application Firewall)
    - 웹 애플리케이션 또는 API에 전송되는 HTTP/HTTPS 웹 트래픽을 모니터링하여 SQL 주입
    - XSS 등과 같은 일반적인 웹 취약점 및 봇으로부터 보호하는 데 특화된 관리형 서비스
    - WAF는 애플리케이션 로드 밸런서 (ALB), CloudFront, API Gateway와 연동되어 작동하며, 별도의 인프라를 구축하거나 관리할 필요가 없으므로 운영 오버헤드가 최소화됨.
  - RDS 파라미터 그룹
    - 데이터베이스 엔진의 런타임 설정을 관리하는 데 사용. 이 그룹을 사용하여 데이터베이스 보안 강화와 관련된 특정 설정을 구성할 수 있음.
    - AWS에서 관리형 서비스로 제공되므로, 보안 설정 관리를 위해 RDS 파라미터 그룹을 사용하는 것은 다른 솔루션(EC2에서 데이터베이스 호스팅)보다 운영 오버헤드가 훨씬 적음.
- C
  - AWS 네트워크 방화벽 (AWS Network Firewall)
    - VPC 내의 네트워크 트래픽(Layer 3-7) 에 대한 방화벽 및 침입 방지 기능을 제공
    - 강력한 보안을 제공하지만, 주로 VPC 경계 또는 세그먼트 간 방어를 위해 사용
    - 웹 애플리케이션 방어에 특화된 WAF보다 구성 및 관리에 더 많은 운영 오버헤드가 발생할 수 있음
- D
  - 최소 권한 원칙을 강화하는 좋은 보안 모범 사례이지만 SQL 주입 공격 자체를 방어하는 솔루션은 아님
  - 코드가 취약하면 권한이 분리되어 있더라도 공격자는 여전히 해당 기능의 계정 권한 내에서 악의적인 명령을 실행할 수 있음.
  - 여기선, 최소 권한 원칙을 사용하고 있는 상황에서 추가적인 웹 공격 방어 메커니즘을 요구함


# 5번
## 키워드
- 실시간 스트리밍 애플리케이션
- 데이터 수집 후, 작업 실행되고 완료되는데 30분 소요
- 워크로드는 들어오는 데이터양이 많아 자주 높은 지연 시간 있음
- 확장가능한 서버리스 솔루션 필요

정답 > A, B

## 근거
- A
  - Amazon Kinesis Data Firehose
    - 서버리스 방식으로 대량의 스트리밍 데이터를 수집하고 Amazon S3나 Redshift와 같은 목적지로 전송하는 데 사용
    - 데이터 양이 많아 지연 시간이 발생하는 상황에서 Firehose는 높은 확장성과 자동 관리를 통해 수집 부하를 처리
    - 필요에 따라 데이터를 버퍼링하고 변환하여 다음 처리 단계에 넘길 수 있음, 이는 서버리스 수집 계층으로 가장 적합함.

- B
  - AWS Lambda
    - 서버리스 컴퓨팅을 제공하며, 수집된 데이터를 변환하거나 분석하는 데 필요한 코드를 실행 
    - 기본적으로 높은 확장성을 제공하므로 대량의 데이터에 유연하게 대응할 수 있음
  - AWS Step Functions
    - 오래 실행되는 워크플로우를 조정(Orchestrate)하는 데 사용됨.
    - Lambda는 실행 시간에 제한(최대 15분)이 있지만, Step Functions는 여러 Lambda 함수를 순차적 또는 병렬적으로 연결하고, 장기 실행 작업을 관리하여 전체 데이터 처리 로직을 안정적으로 완료할 수 있게 함.
    - 이 조합 (Lambda + Step Functions)은 성능을 향상시키고, 지연 시간을 줄이며, 필요한 확장성과 서버리스 요구 사항을 모두 만족


- C
  - AWS Database Migration Service (AWS DMS)
    - 데이터베이스를 마이그레이션하거나 데이터를 복제하는 도구

- D
  - EC2 인스턴스는 서버 관리가 필요하므로 서버리스 요구 사항을 충족하지 못함
  - 운영 오버헤드가 증가

- E
  - Fargate
    - 컨테이너에 대한 서버리스 컴퓨팅을 제공하여 서버리스 요구 사항을 충족할 수 있음
    - 하지만, 스트리밍 데이터를 수집하고 오래 실행되는 처리 워크플로우를 조정하는 데 있어서는 Kinesis Firehose와 (Lambda + Step Functions)의 조합이 AWS 스트리밍 및 서버리스 아키텍처에서 가장 효율적이고 일반적인 패턴
    - 특히 30분이라는 긴 처리 시간을 안정적으로 관리하기 위해서는 Step Functions가 필수적
