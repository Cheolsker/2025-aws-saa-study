
# 1번
## 요구사항
- AWS로 마이그레이션해야 하는 Windows 기반 애플리케이션이 여러 가용성 영역(AZ)에 배포된 여러 Amazon EC2 Windows 인스턴스에 연결된 공유 Windows 파일 시스템을 사용해야 한다

## 정답
- B

EFS -> 여러 AZ의 파일시스템을 확장/축소할 수 있지않음?
EBS -> EBS는 EC2 인스턴스마다 연결하는 거 아님?
  스냅샷을 찍어야 다른 AZ에 복제할 수 있음. S3에 저장.

## 근거
- A ❌
  - "볼륨 게이트웨이 모드(Volume Gateway mode)"는 AWS Storage Gateway 서비스의 세 가지 주요 유형(Volume, File, Tape) 중 하나로, 주로 온프레미스(On-premises) 환경의 애플리케이션에 iSCSI(Internet Small Computer System Interface) 블록 스토리지를 제공하면서, 그 데이터를 클라우드(Amazon S3)에 백업 또는 저장할 수 있도록 하는 하이브리드 클라우드 스토리지 솔루션

- B ✅
  - Amazon FSx for Windows File Server는 AWS 클라우드에서 완전히 관리되는 확장 가능하고 안정적이며 안전한 네이티브 Microsoft Windows 파일 시스템을 제공
  - Windows Server의 모든 기능(예: SMB 프로토콜 지원, NTFS, Active Directory 통합)을 지원
  - 표준 SMB 프로토콜을 사용하여 여러 AZ에 걸쳐 배포된 Windows EC2 인스턴스가 공유 파일 시스템에 동시에 액세스할 수 있음

- C ❌
  - Amazon EFS는 주로 `Linux 기반 파일 시스템(NFS 프로토콜 사용)용` 으로 설계
  - Windows EC2 인스턴스는 EFS를 마운트할 수 있지만, 기본적으로 Windows 기반 애플리케이션에 필요한 `네이티브 SMB 지원 및 NTFS 권한`을 제공하지 않음

- D 
  - Amazon EBS는 블록 스토리지이며, `단일 EC2 인스턴스에 연결되도록 설계`됨.
  - 기본적으로 여러 EC2 인스턴스(특히 여러 AZ에 걸쳐)에 동시에 마운트되는 공유 파일 시스템을 지원하지 않음.



# 2번

## 정답
- C

## 근거
- A ❌
  - `Amazon Inspector`
    - EC2 인스턴스 또는 컨테이너 이미지의 보안 취약점을 평가하고 감지하는 서비스
    - 실시간 DDoS 공격을 감지하거나 완화하는 도구가 아님.
    - ALB에는 Inspector 에이전트를 설치할 수도 없음.

- B ❌
  - `Amazon Macie`
    - Amazon S3에 저장된 데이터에서 **민감한 정보(예: 개인 식별 정보)**를 식별하고 보호하는 데 중점을 둔 데이터 보안 및 개인 정보 보호 서비스

- C ✅
  - `AWS Shield Standard`
    - 모든 AWS 고객에게 자동으로 제공되어 가장 일반적인 `네트워크 및 전송 계층 DDoS 공격`으로부터 보호
  - `AWS Shield Advanced`
    - 표준 보호 이상의 기능이 필요할 때 사용
    - Shield Advanced 는 추가 비용을 내고 추가적인 서비스를 제공받는 것으로 L7 트래픽 모니터링, 사후 분석 등의 기능을 제공함
    - EC2, ELB, Cloudfront, Global Accelerator, Route 53 와 연동된 애플리케이션에 대한 추가적인 보호 가능
    - **AWS의 전담 DDoS 대응팀(DRT)**의 지원을 받음 - 연중무휴(24/7) 전문 엔지니어의 상시 지원

- D ❌
  - `Amazon GuardDuty`
    - `AWS 계정과 워크로드를 보호하는 지능형 위협 탐지 서비스`
    - 비정상적인 API 호출, 무단 액세스 등 잠재적인 보안 위협을 모니터링하고 경고 
    - 계정/인프라 보안 침해(행동의 질적 위협)를 탐지하는 데 중점
      - 예시)
      - 해외 IP에서 수많은 인스턴스를 시작하려는 시도
      - EC2 인스턴스가 암호화폐 채굴에 사용되거나, 승인되지 않은 포트에서 통신하는 등 비정상적인 동작을 보이는 경우
      - EC2 인스턴스에 대한 무차별 대입 공격(Brute-force attack) 시도.
      - S3 버킷에 대한 비정상적이거나 의심스러운 접근 패턴.



# 3번

## 정답
- A

## 근거
- A ✅
  - `Amazon GuardDuty`
    - AWS 계정, 워크로드, 저장된 데이터에서 악의적인 활동과 무단 동작을 지속적으로 모니터링하고 지능적으로 위협을 탐지하는 서비스
  - `EventBridge`
    - 특정 기준(예: 심각도가 높은 특정 위협 유형)에 따라 이 위협 결과를 필터링하고, 미리 정의된 AWS Lambda 함수를 자동으로 호출하여 대응 조치를 실행하도록 구성할 수 있음.
  - `Lambda 함수`
    - 공격 원천 IP 주소를 추출하고, 이를 이미 존재하는 AWS WAF에 새 규칙(예: 해당 IP를 차단하는 규칙)으로 자동으로 추가하여 웹 애플리케이션을 보호할 수 있음

- B ❌
  - `AWS Firewall Manager`
    - 여러 계정과 VPC에 걸쳐 AWS WAF 규칙이나 Shield Advanced 설정과 같은 보안 구성을 중앙에서 관리 및 배포하는 서비스. 
    - 이는 위협 탐지 서비스가 아님.

- C ❌
  - `Amazon Inspector`
    - EC2 인스턴스, 컨테이너 이미지 등에서 소프트웨어 취약점 및 의도하지 않은 네트워크 노출을 평가하고 감지하는 서비스
    - 이는 실시간 악의적 활동 및 행동 이상을 탐지하는 GuardDuty의 기능과는 다름.

- D ❌
  - `Amazon Macie`
    - S3 버킷 내의 데이터에서 **민감한 정보(PII 등)**를 탐지하는 데이터 보안 서비스
    - 웹 애플리케이션의 행동 기반 위협을 탐지하는 목적과는 전혀 관련이 없음.

- 기타 🌟
  - `WAF 웹 ACL (Web Access Control List)`
    - 의미 
      - 웹 애플리케이션에 대한 HTTP/HTTPS 요청을 분석하고, 사용자가 정의한 조건에 따라 해당 요청을 허용, 차단 또는 카운트하도록 지시하는 규칙 모음
    - 작동 위치
      - Amazon CloudFront, Application Load Balancer (ALB), Amazon API Gateway, AWS AppSync와 같은 엣지(Edge) 서비스 앞에 연결
    - 제어 내용
      - 특정 IP 주소 또는 범위.
      - 요청 헤더(User-Agent 등).
      - HTTP 메서드 (GET, POST 등).
      - 요청 본문이나 쿼리 문자열에서 SQL Injection이나 **교차 사이트 스크립팅(XSS)**과 같은 악성 패턴.
  
  - `VPC 네트워크 ACL (Network Access Control List)`
    - 의미
      - 서브넷 수준에서 인바운드(Inbound) 및 아웃바운드(Outbound) 트래픽을 제어하는 무상태(Stateless) 규칙 목록
    - 작동 위치
      - 서브넷 경계에 연결됩니다. 서브넷 내부의 모든 인스턴스에 적용
    - 제어하는 내용
      - 기본 네트워크 정보만을 사용하여 트래픽을 제어합니다.
      - 프로토콜 (TCP, UDP, ICMP)
      - 출발지/목적지 IP 주소 및 CIDR 범위
      - 출발지/목적지 포트 번호.
    - 특징 (무상태)
      - 들어오는 트래픽을 허용(Allow)하는 규칙을 설정했다면, 해당 트래픽에 대한 응답(나가는 트래픽) 역시 명시적으로 허용하는 규칙을 만들어야 합니다.
    - 목적
      - 서브넷 수준에서 트래픽의 허용/거부를 최종적으로 결정하여 VPC 내의 인스턴스에 대한 기본적인 네트워크 보안 계층을 제공합니다.



# 4번
## 정답
- C

## 근거
- A ❌
  - `NLB (Network Load Balancer)`
    - 레이어 4(전송 계층)에서 작동. 매우 빠른 성능과 고정 IP 주소를 제공하지만, 쿼리 문자열 기반 라우팅과 같은 고급 레이어 7 기능을 제공하지 않음

- B ❌
  - `GWLB (Gateway Load Balancer)`
    - 네트워크 가상 어플라이언스(방화벽, 침입 탐지 시스템 등)를 투명하게 배포, 확장 및 관리하는 데 사용
    - 트래픽 라우팅 기능이 일반 로드 밸런서와 다르며 쿼리 문자열 기반 라우팅을 지원하지 않음.

- C ✅
  - `ALB (Application Load Balancer)`
    - 레이어 7(애플리케이션 계층)에서 작동하는 퍼블릭 로드 밸런서로, 웹 트래픽을 처리하고 백엔드 서비스(예: EC2 인스턴스, 컨테이너 등)로 라우팅하는 데 적합
    - **쿼리 문자열 매개변수 기반 라우팅(Query String Parameter Based Routing)**을 지원. 이 기능을 사용하여 수신되는 요청의 쿼리 문자열에 특정 값이 포함되어 있는지 여부에 따라 트래픽을 서로 다른 대상 그룹(Target Group)으로 라우팅할 수 있음
    - **AWS Certificate Manager (ACM)**에서 발급하거나 가져온 인증서를 리스너에 연결하여 SSL/TLS 오프로딩을 수행할 수 있습니다. 이는 로드 밸런서가 암호화된 트래픽을 해독하고 백엔드에는 비암호화되거나 재암호화된 트래픽을 전달하여 보안을 유지하는 동시에 백엔드 서버의 부담을 줄여줍니다

- D 
 - A와 마찬가지로 NLB는 쿼리 문자열 기반 라우팅 지원안함 



# 5번
## 정답
- B

## 근거

- A ❌
  - `CloudFront`
    - CDN(콘텐츠 전송 네트워크) 서비스로, 일반적으로 최종 사용자에게 콘텐츠를 배포하는 데 사용

- B ✅
  - `Global Accelerator`
    - AWS 글로벌 네트워크를 활용
    - 사용자의 트래픽은 가장 가까운 **AWS 엣지 로케이션(Edge Location)**으로 라우팅됩니다. 엣지 로케이션에서 트래픽은 공용 인터넷을 우회하여 AWS의 안정적이고 혼잡이 적은 프라이빗 백본 네트워크를 통해 대상(방송 시스템)으로 이동
    - TCP 핸드셰이크 및 데이터 전송을 가속화하여, 특히 장거리에서 발생하는 대기 시간, 지터 및 패킷 손실을 크게 줄여줍니다. 이는 실시간 스트리밍(RTMP) 품질을 보장하는 데 매우 중요
    - Global Accelerator는 레이어 4(L4) 서비스로, TCP 및 UDP 트래픽을 처리할 수 있습니다. RTMP는 기본적으로 TCP를 사용하므로 Global Accelerator를 통해 효율적으로 가속화
    - 두 개의 고정된 글로벌 애니캐스트 IP 주소를 제공하여, 전 세계 모든 기자들이 동일한 IP 주소로 트래픽을 보낼 수 있게 함

- C 
  - `AWS 클라이언트 VPN (AWS Client VPN)`
    - VPN은 원격 사용자가 사설 네트워크에 안전하게 액세스하도록 하는 데 중점
    
- D
  - Amazon EC2 인스턴스 및 AWS Elastic IP 주소
    - 기본적인 컴퓨팅 자원과 고정 IP를 제공하지만, 전 세계적인 TCP 연결 가속화를 위한 AWS 글로벌 네트워크의 이점은 제공하지 않음. 
    - 트래픽은 여전히 혼잡한 공용 인터넷을 통해 전송되어 품질 문제가 발생할 수 있음.
